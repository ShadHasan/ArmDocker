### create separate build directory

# Create build directory
mkdir -p build && cd build

### Configuration generation:
$> export OPENCV_DOWNLOAD_PATH=/tmp/opencv-cache
$> cmake -DWITH_OPENCL=ON ../opencv-4.x/

""" Note: opencv_contrib

Note opencv_contrib does not work may be because of python dependecies which we don't want anything python related.
-DOPENCV_EXTRA_MODULES_PATH=../opencv_contrib-4.x/
"""

"""Note: package config 
-DOPENCV_GENERATE_PKGCONFIG=ON 

is not supported from 4.x version
"""

""" Notes for cache
Configuration script can try to download additional libraries and files from the internet, if it fails to do it corresponding features will be turned off. In some cases configuration error can occur. By default all files are first downloaded to the <source>/.cache directory and then unpacked or copied to the build directory. It is possible to change download cache location by setting environment variable or configuration option:

export OPENCV_DOWNLOAD_PATH=/tmp/opencv-cache
cmake ../opencv
# or
cmake -DOPENCV_DOWNLOAD_PATH=/tmp/opencv-cache ../opencv
"""


### Build

$> cmake build .

Equivalent we can use "make" to build

$> make -j4

where "-j" is to say amount of processor to be ustilised


### Installation

$> make install

By default OpenCV will be installed to the /usr/local directory, all files will be copied to following locations:

/usr/local/bin - executable files
/usr/local/lib - libraries (.so)
/usr/local/cmake/opencv4 - cmake package
/usr/local/include/opencv4 - headers
/usr/local/share/opencv4 - other files (e.g. trained cascades in XML format)
Since /usr/local is owned by the root user, the installation should be performed with elevated privileges (sudo):

#include <iostream>
#include <fstream>
#include <string>
#include <iterator>
#include <opencv2/opencv.hpp>
#include <opencv2/core/ocl.hpp>

using namespace std; // Brings all names from std into the global scope

int main()
{
  // Get the default context
  cv::ocl::Context context = cv::ocl::Context::getDefault();
  if (!cv::ocl::haveOpenCL())       
      cout << "OpenCL is not avaiable..." << endl;          
   else cout << "OpenCL is AVAILABLE! :) " << endl; //this is the output

   cv::ocl::setUseOpenCL(true);

   cout << context.ndevices() << " GPU devices are detected." << endl; 
   for (int i = 0; i < context.ndevices(); i++)
   {
     cv::ocl::Device device = context.device(i);
     cout << "name:              " << device.name() << endl;
     cout << "available:         " << device.available() << endl;
     cout << "imageSupport:      " << device.imageSupport() << endl;
     cout << "OpenCL_C_Version:  " << device.OpenCL_C_Version() << endl;
     cout << endl;
    } //this works & i can see my video card name & opencl version
    cv::ocl::Device(context.device(0));
}

## Linking and building shared object c class

Linking a shared object in C involves two primary steps using a compiler like GCC on Linux: compile-time linking (telling the compiler about the library's symbols) and runtime loading (telling the operating system where to find the actual file when the program runs). 

#### Step 1: Create the Shared Library 

First, you need to create the shared object file (.so on Linux). 

Write the library code and header file:
library.h: Contains function prototypes.
library.c: Contains the function definitions.

Compile the source files into Position Independent Code (PIC):

```bash
gcc -c -fPIC library.c -o library.o
```
Create the shared library file:

```bash
gcc -shared -o libmylibrary.so library.o
```

By convention, shared libraries are named lib prefix and a .so suffix (e.g., libmylibrary.so), so the linker can easily find them later. 

#### Step 2: Compile and Link Your Program
Next, compile your main program and link it against the shared library. 

Write the main program (main.c): This file must #include "library.h".

Compile the main program and link it:

```bash
gcc main.c -L. -lmylibrary -o myprogram
```
-L. tells the linker to look for libraries in the current directory (.) in addition to standard locations.
-l followed by the library name (without the lib prefix or .so suffix, e.g., mylibrary for libmylibrary.so) specifies which library to link against. 

#### Step 3: Ensure the Library is Found at Runtime 
When you run your program, the operating system's dynamic loader needs to locate the shared library file. If it's not in a standard system directory (like /usr/lib), you'll get an error. 

You can solve this in two common ways:

Set the LD_LIBRARY_PATH environment variable (temporary):
```bash
export LD_LIBRARY_PATH=.
```

```executable
./myprogram
```

(Note: The export command is for Bash/sh; use setenv LD_LIBRARY_PATH . for Tcsh).

Embed the path using rpath (recommended): This embeds the library location directly into the executable at link time.

```bash
gcc main.c -L. -Wl,-rpath=. -lmylibrary -o myprogram

./myprogram
```

For a comprehensive guide, resources like Cprogramming.com offer detailed tutorials on creating and using shared libraries in C.

# How to use CV with OpenCL that is ocl::cv(From Google AI) 
### Note: Partially Explored


OpenCV's cv::ocl module allows you to write custom OpenCL kernels and interact with the OpenCL context and devices directly. However, a more common and recommended approach is to use the T-API (Transparent API) which automatically utilizes OpenCL acceleration for standard OpenCV functions when available, by using cv::UMat instead of cv::Mat. 


Using the Transparent API (T-API) 
The T-API is the easiest way to leverage OpenCL acceleration. Simply replace cv::Mat with cv::UMat in your code. 

```cpp code
#include <opencv2/opencv.hpp>
#include <iostream>

int main() {
    // Enable OpenCL transparent API
    if (!cv::ocl::haveOpenCL()) {
        std::cout << "OpenCL not available. Using CPU fallback." << std::endl;
    } else {
        cv::ocl::setUseOpenCL(true);
        std::cout << "OpenCL is available and enabled." << std::endl;
    }

    cv::Mat cpu_img; // Regular CPU Mat
    // Load image into cpu_img
    // (e.g., cpu_img = cv::imread("lena.png", cv::IMREAD_GRAYSCALE);)

    // Use UMat for automatic OpenCL acceleration
    cv::UMat gpu_img, blurred_gpu_img;

    // Data is automatically transferred to GPU memory
    gpu_img = cpu_img.getUMat(cv::ACCESS_READ);

    // This operation will run on the GPU if OpenCL is enabled
    cv::GaussianBlur(gpu_img, blurred_gpu_img, cv::Size(5, 5), 0);

    // Data is automatically transferred back to CPU memory when needed
    cv::Mat output_cpu = blurred_gpu_img.getMat(cv::ACCESS_READ);

    // Display or save the result
    // cv::imshow("Blurred GPU", output_cpu);
    // cv::waitKey(0);

    return 0;
}
```

### Writing a Custom OpenCL Kernel with cv::ocl 

For situations where you need to write a custom OpenCL kernel, you can use classes like cv::ocl::Context, cv::ocl::ProgramSource, cv::ocl::Program, and cv::ocl::Kernel.

Here is a simplified example demonstrating the key steps involved, based on snippets from the OpenCV documentation and community examples:

Initialize OpenCL Context and Device:
```cpp
#include <opencv2/core/ocl.hpp>
#include <iostream>

if (!cv::ocl::haveOpenCL()) {
    std::cerr << "OpenCL not available." << std::endl;
    return -1;
}
cv::ocl::Context context;
if (!context.create(cv::ocl::Device::TYPE_GPU)) {
    std::cerr << "Failed to create OpenCL context." << std::endl;
    return -1;
}
cv::ocl::Device device = context.device(0); // Select the first device
std::cout << "Using device: " << device.name() << std::endl;
```

Define the OpenCL Kernel Source (as a C++ string):
```cpp
const char* kernelSource = "__kernel void myKernel(__global const uchar* src, __global uchar* dst, int cols, int rows) {\n"
                           "    int x = get_global_id(0);\n"
                           "    int y = get_global_id(1);\n"
                           "    if (x < cols && y < rows) {\n"
                           "        dst[y * cols + x] = 255 - src[y * cols + x]; // Invert pixel value\n"
                           "    }\n"
                           "}\n";
```

Build the Program and Kernel:

```cpp
cv::ocl::ProgramSource programSource(kernelSource);
std::string errorMessage;
cv::ocl::Program program = context.buildProgram(programSource, "", errorMessage);
if (!errorMessage.empty()) {
    std::cerr << "Build error: " << errorMessage << std::endl;
    return -1;
}
cv::ocl::Kernel kernel("myKernel", program);
```

Prepare Data (UMat) and Execute the Kernel:

cpp
```cv::Mat mat_src = cv::imread("lena.png", cv::IMREAD_GRAYSCALE);
cv::UMat u_src, u_dst;
mat_src.copyTo(u_src); // Copy data to UMat (GPU memory)
u_dst.create(mat_src.size(), mat_src.type());

// Set kernel arguments
kernel.set(0, u_src);
kernel.set(1, u_dst);
kernel.set(2, mat_src.cols);
kernel.set(3, mat_src.rows);

// Define work-group dimensions
size_t globalSize[2] = {(size_t)mat_src.cols, (size_t)mat_src.rows};
size_t localSize[2] = {16, 16}; // Optimal local size may vary by device

// Run the kernel
kernel.run(2, globalSize, localSize, false);
cv::ocl::finish(); // Wait for completion

// Get the result back to CPU Mat
cv::Mat mat_dst;
u_dst.copyTo(mat_dst);

// Display or save
// cv::imshow("Inverted Custom Kernel", mat_dst);
// cv::waitKey(0);
```
